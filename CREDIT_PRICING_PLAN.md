# Qritor 积分计费方案

基于 Vercel AI Gateway 真实模型价格制定。

---

## 一、核心参数

| 参数 | 值 | 说明 |
|------|------|------|
| **积分兑换比例** | 1 USD = 100 积分 | 用户购买时的售价比例 |
| **加价倍率** | ×2 | 保证 50% 利润率 |
| **最低消耗** | 1 积分 | 每次调用最少扣 1 积分 |

### 计费公式

```
积分消耗 = ceil( (inputTokens × inputPricePerM + outputTokens × outputPricePerM) / 1,000,000 × 200 )
                                                                                                 ↑
                                                                         = 100（积分/美元） × 2（加价倍率）
```

**拆解**：

1. `inputTokens × inputPricePerM / 1,000,000` → 输入的 API 成本（美元）
2. `outputTokens × outputPricePerM / 1,000,000` → 输出的 API 成本（美元）
3. 两者相加 → 本次调用的总 API 成本（美元）
4. `× 100` → 按售价比例换算成积分
5. `× 2` → 加价倍率，保证 50% 利润
6. `ceil()` → 向上取整，最少 1 积分

**利润验证**：用户花 $20 买 2000 积分，这 2000 积分最多消耗 $10 的 API → 利润 $10，利润率 50%。

## 五、模型选择器倍率展示

桌面端模型选择器需要给用户直观展示不同模型的价格差异。

### 倍率定义

以最便宜的模型（GLM-5）为基准 `1×`，其余模型按续写场景（20K input + 3K output）的积分消耗计算倍率：

| 模型 | 积分/次 | 倍率 | 展示标签 |
|------|--------|------|---------|
| zai/glm-5 | 1 | 1× | `1×` |
| zai/glm-4.7-flash | 1 | 1× | `1×` |
| minimax/minimax-m2.5 | 2 | 2× | `2×` |
| google/gemini-3-flash | 3 | 3× | `3×` |
| moonshotai/kimi-k2.5 | 7 | 7× | `7×` |
| deepseek/deepseek-v3.2-thinking | 11 | 11× | `11×` |
| xai/grok-4.1-fast-reasoning | 16 | 16× | `16×` |
| anthropic/claude-opus-4.6 | 21 | 21× | `21×` |
| anthropic/claude-sonnet-4.5 | 21 | 21× | `21×` |

### 前端展示方案

在模型选择器中，模型名称右侧显示倍率标签，用颜色区分等级：

```
┌────────────────────────────────────────┐
│  GLM-5                         1×  🟢 │
│  GLM-4.7 Flash                 1×  🟢 │
│  MiniMax M2.5                  2×  🟢 │
│  Gemini 3 Flash                3×  🟡 │
│  Kimi K2.5                     7×  🟡 │
│  DeepSeek V3.2 Thinking       11×  🟠 │
│  Grok 4.1 Fast Reasoning      16×  🟠 │
│  Claude Opus 4.6              21×  🔴 │
│  Claude Sonnet 4.5            21×  🔴 │
└────────────────────────────────────────┘
```

**颜色规则**：
- 🟢 绿色（1×-3×）：经济型
- 🟡 黄色（4×-10×）：标准型
- 🟠 橙色（11×-20×）：高级型
- 🔴 红色（21×+）：旗舰型

### 倍率计算方式

倍率存储在 `llm_config` 表的 `price_multiplier` 字段（INT），管理员在后台设置。

计算参考公式（用于管理员参考，非前端实时计算）：
```
price_multiplier = round(
  (inputPricePerM × 20000 + outputPricePerM × 3000) /
  (0.10 × 20000 + 0.40 × 3000)
)
```
即：以 GLM-5 为基准，按续写场景（20K input + 3K output）的成本比值取整。

## 六、图片/视频生成定价

图片和视频不走 token 计费，使用固定积分（通过 `image_gen_config.credit_rate` / `video_gen_config.credit_rate` 配置）。

### 图片生成

```
积分消耗 = ceil(width × height / 1,000,000 × credit_rate)
```

| 模型等级 | credit_rate | 1024×1024 | 2048×2048 |
|---------|------------|-----------|-----------|
| 标准 | 8 | 9 积分 | 34 积分 |
| 高级 | 15 | 16 积分 | 63 积分 |
| 顶级 | 25 | 27 积分 | 105 积分 |

### 视频生成

```
积分消耗 = ceil(duration × credit_rate)
```

| 模型等级 | credit_rate | 5 秒 | 10 秒 |
|---------|------------|------|-------|
| 标准 | 20 | 100 积分 | 200 积分 |
| 高级 | 40 | 200 积分 | 400 积分 |
| 顶级 | 80 | 400 积分 | 800 积分 |

## 八、实施清单

### 数据库修改

**llm_config 表**：
- 新增 `input_price_per_m` DECIMAL(10,6) — 输入价格（$/1M tokens）
- 新增 `output_price_per_m` DECIMAL(10,6) — 输出价格（$/1M tokens）
- 新增 `price_multiplier` INT — 价格倍率（前端展示用，以 GLM-5 为基准 1×）
- 移除 `normalization_factor` — 不再使用
- 移除 `credit_rate` — 不再使用

**credit_transactions 表**：
- 新增 `input_price_per_m` FLOAT — 记录当时的输入价格
- 新增 `output_price_per_m` FLOAT — 记录当时的输出价格
- 移除 `credit_rate` — 不再使用
- 移除 `normalization_factor` — 不再使用

### 代码修改

1. `token-calculator.ts` — 新计费公式（×200）
2. `credit-service.ts` — 交易记录字段替换
3. `route.ts`（LLM/图片/视频） — 适配新参数
4. Prisma Schema — 字段增删
5. 管理后台 — LLM 配置表单添加价格字段

### 配置数据

各模型的 `input_price_per_m` 和 `output_price_per_m` 需要写入 `llm_config` 表。
