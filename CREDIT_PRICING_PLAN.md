# Qritor 积分计费方案

基于 Vercel AI Gateway 真实模型价格制定。

---

## 一、核心参数

| 参数 | 值 | 说明 |
|------|------|------|
| **积分兑换比例** | 1 USD = 100 积分 | 用户购买时的售价比例 |
| **加价倍率** | ×2 | 保证 50% 利润率 |
| **最低消耗** | 1 积分 | 每次调用最少扣 1 积分 |

### 计费公式

```
积分消耗 = ceil( (inputTokens × inputPricePerM + outputTokens × outputPricePerM) / 1,000,000 × 200 )
                                                                                                 ↑
                                                                         = 100（积分/美元） × 2（加价倍率）
```

**拆解**：

1. `inputTokens × inputPricePerM / 1,000,000` → 输入的 API 成本（美元）
2. `outputTokens × outputPricePerM / 1,000,000` → 输出的 API 成本（美元）
3. 两者相加 → 本次调用的总 API 成本（美元）
4. `× 100` → 按售价比例换算成积分
5. `× 2` → 加价倍率，保证 50% 利润
6. `ceil()` → 向上取整，最少 1 积分

**利润验证**：用户花 $20 买 2000 积分，这 2000 积分最多消耗 $10 的 API → 利润 $10，利润率 50%。

## 四、订阅套餐

| 套餐 | 售价 | 积分 | 最大 API 成本 | 利润率 |
|------|------|------|-------------|--------|
| **FREE** | $0/月 | 100 | $0.50 | 获客成本 |
| **PLUS** | $20/月 | 2,000 | $10 | **50%** |
| **PRO** | $60/月 | 6,000 | $30 | **50%** |
| **MAX** | $200/月 | 20,000 | $100 | **50%** |

### 各套餐典型使用场景

**FREE（100 积分/月）** — 试用体验
- 用 Grok/DeepSeek 续写 50 次，或用 Claude Sonnet 续写 4 次
- 目标：让用户体验产品，引导付费

**PLUS（2,000 积分/月）** — 日常创作
- 每天用 Grok/DeepSeek 续写 33 次 → 月消耗 ~2,000 积分
- 每天用 Claude Sonnet 续写 3 次 → 月消耗 ~1,890 积分
- 目标：中度创作用户

**PRO（6,000 积分/月）** — 专业创作
- 每天用 Claude Sonnet 续写 9 次 → 月消耗 ~5,670 积分
- 混合使用：大纲用 Grok + 正文用 Claude → 每天产出更多
- 目标：重度创作者、多项目并行

**MAX（20,000 积分/月）** — 极限创作
- 每天用 Claude Sonnet 续写 30 次 → 月消耗 ~18,900 积分
- 每天用 Claude Opus 续写 18 次 → 月消耗 ~18,900 积分
- 目标：专业团队、商业用户

## 五、模型选择器积分展示

桌面端模型选择器需要给用户直观展示不同模型的积分消耗。

### 展示方案

模型名称右侧显示"每次续写大约消耗多少积分"，用颜色区分价格等级：

```
┌─────────────────────────────────────────────┐
│  Grok 4.1 Fast Reasoning    ~2 积分/次  🟢  │
│  GLM-4.7 Flash              ~2 积分/次  🟢  │
│  DeepSeek V3.2 Thinking     ~2 积分/次  🟢  │
│  MiniMax M2.5               ~2 积分/次  🟢  │
│  Kimi K2.5                  ~4 积分/次  🟢  │
│  Gemini 3 Flash             ~4 积分/次  🟢  │
│  GLM-5                      ~6 积分/次  🟡  │
│  Claude Sonnet 4.5         ~21 积分/次  🔴  │
│  Claude Opus 4.6           ~35 积分/次  🔴  │
└─────────────────────────────────────────────┘
```

**颜色规则**：
- 🟢 绿色（1~5 积分）：经济型
- 🟡 黄色（6~10 积分）：标准型
- 🟠 橙色（11~20 积分）：高级型
- 🔴 红色（21+ 积分）：旗舰型

### 数据来源

`llm_config` 表的 `estimated_credits_per_use` 字段（INT），管理员在后台设置。

参考计算方式（按续写场景 20K input + 3K output）：
```
estimated_credits_per_use = ceil(
  (inputPricePerM × 20000 + outputPricePerM × 3000) / 1,000,000 × 200
)
```

## 七、图片/视频生成定价

图片和视频不走 token 计费，使用固定积分（通过 `image_gen_config.credit_rate` / `video_gen_config.credit_rate` 配置）。

### 图片生成

```
积分消耗 = ceil(width × height / 1,000,000 × credit_rate)
```

| 模型等级 | credit_rate | 1024×1024 | 2048×2048 |
|---------|------------|-----------|-----------|
| 标准 | 8 | 9 积分 | 34 积分 |
| 高级 | 15 | 16 积分 | 63 积分 |
| 顶级 | 25 | 27 积分 | 105 积分 |

### 视频生成

```
积分消耗 = ceil(duration × credit_rate)
```

| 模型等级 | credit_rate | 5 秒 | 10 秒 |
|---------|------------|------|-------|
| 标准 | 20 | 100 积分 | 200 积分 |
| 高级 | 40 | 200 积分 | 400 积分 |
| 顶级 | 80 | 400 积分 | 800 积分 |

## 八、实施清单

### 数据库修改

**llm_config 表**：
- 新增 `input_price_per_m` DECIMAL(10,6) — 输入价格（$/1M tokens）
- 新增 `output_price_per_m` DECIMAL(10,6) — 输出价格（$/1M tokens）
- 新增 `estimated_credits_per_use` INT — 每次续写预估积分消耗（前端展示用）
- 移除 `normalization_factor` — 不再使用
- 移除 `credit_rate` — 不再使用

**credit_transactions 表**：
- 新增 `input_price_per_m` FLOAT — 记录当时的输入价格
- 新增 `output_price_per_m` FLOAT — 记录当时的输出价格
- 移除 `credit_rate` — 不再使用
- 移除 `normalization_factor` — 不再使用

### 代码修改

1. `token-calculator.ts` — 新计费公式（×200）
2. `credit-service.ts` — 交易记录字段替换
3. `route.ts`（LLM/图片/视频） — 适配新参数
4. Prisma Schema — 字段增删
5. 管理后台 — LLM 配置表单添加价格字段

### 配置数据

各模型的 `input_price_per_m` 和 `output_price_per_m` 需要写入 `llm_config` 表。
